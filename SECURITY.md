# Security Policy

## Supported Versions

This project currently supports security fixes for the latest `main` branch only.

## Reporting a Vulnerability

Please do **not** open public GitHub issues for security vulnerabilities.

Use one of the following private channels:

1. GitHub Security Advisories (preferred):
   - Go to the repository Security tab
   - Click "Report a vulnerability"
2. Or contact the maintainer directly if Security Advisories are unavailable.

## What to Include

Please include:

- A clear description of the issue
- Affected version/commit
- Reproduction steps or proof-of-concept
- Potential impact
- Suggested fix (if available)

## Response Targets

- Initial response: within 7 days
- Fix or mitigation plan: as soon as reasonably possible

## Disclosure

Please allow time for a fix before public disclosure.
After a fix is released, coordinated disclosure is welcome.
